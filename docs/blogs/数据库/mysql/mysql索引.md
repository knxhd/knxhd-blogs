## 数据结构

### 二叉树

#### 特点：

>1. 父节点下有2个子节点，即左子节点和右子节点
>2. 左子节点值比父节点小，右子节点比父节点大
>3. 可使用二分法查找方式，查询效率较快

- 不足之处

>1. 可能造成二叉树的深度较大

### 平衡二叉树

#### 特点

>1. 父节点下有2个子节点，即左子节点和右子节点
>2. 左子节点值比父节点小，右子节点比父节点大
>3. 右子节点深度和左子节点深度最大差<=1bbb

#### 优点

>1. 避免二叉树的极端情况，导致IO读取次数较多

#### 缺点

>1. 每个节点只能存储一个数据，存储数据较少，不适合大数据量的操作
>2. 插入、更新和删除时，需要通过旋转操作来保证树的长度平衡，耗时较长(不作为索引的数据结构，并不是主要的原因)

### B-树

#### 特点

>1. 每个节点包含2个数据
>2. 父节点下有2个子节点，即左子节点和右子节点
>3. 左子节点值比父节点小，右子节点比父节点大
>4. 右子节点深度和左子节点深度最大差<=1
>5. 叶节点存储的数据为 索引值和数据指针地址

#### 优点

>1. 存储的数据较多
